Some common block tags

hr

The < hr/> element used to stand for "horizontal line" and this is still how it is typically displayed by browsers.However, in HTML5 the tag took on a semantic meaning. "semantic break".I will be using it often to break up example files when I want to display multiple tags in one file.

address

The < address> element indicates that the text is an address specific to the page, perhaps contact information for the author.

1715 Walant Blwd
Ashabula OH 4400

blockquote

The < blockquote> element is used for extended quotations.It should include the cite attribute in order to show or link to the source of the quote.

If you cannot do great things,do small things in a great way. Napolean Hill

details

The < details> element is used in conjunction with the < summary> tag.It is one of the tags that was created in response to the repeated use of scripting to open and close content.It is not supported by all of the browsers yet though.

The summary

The rest of the details content goes here.If only < details> element is not supported, then the summary and the content are always visible.

Some common inline tags

Why do we bother with inline elements? There are certainly times we want to mark up content, but we don't want to insert the line breaks around them.The < span> tag was the first to give that ability, but it was mostly used to style the code, and we don't want to do that.Some of the newer tags that still separate content and style are listed below.Remember, inline tags can flow next to each other and don't cause a line break before and after the element. We have seen this with links and images.

Citing a source - cite

The < cite> makes it much easier for people and bots to search the code for this specific information.

A great reference for these and other element is https://www.brainyquote.com/authors/napoleon-hill-quotes

Defining your abbreviations - abbr

The < abbr> will display an abbrivation on the browser.But if a title attribute is specified in the tag, the title will appear when the mouse hovers over the abbreviation.Screen readers also provide the title information.

The days of the week are often abbreviated:Sun,Mon,Tues,etc.

Specifying the use of time -time

The < time> is again a completely semantic tag that can be used to search for specific information.

The party starts at 9:00pm.

The party took place on New Year's Eve.

Identifying code fragments - code

The < code> element tends to be displayed in a different font, but also has the semantic meaning to convey information to as many people as possible.

I have been trying to learn how to write html but my img elements never seem to work.

Using subscripts and superscripts - sub and sup

The < sub> and < sup> elements are used to create the super and subscripts, e.g. (x+y)3 or H2O. In the past, people would often paste mathematical images into the page. But images didn't convey the same meaning to all users.
